# RAVO 简短测试用例

这些 prompt 适合在一个全新的 Codex 会话里直接测试，不依赖任何项目上下文。每条都很短，目标是快速验证一项 RAVO 行为，不让 Codex 跑太久。

## 1. 需求分析

Prompt：

```text
我想做一个简单的习惯打卡应用。先不要写代码，先告诉我真正的用户是谁、目标是什么、有哪些关键约束、你推荐哪种方案。
```

预期效果：

- RAVO 应优先进入需求分析，而不是直接开始实现。
- 回答应覆盖目标、消费者、约束、方案取舍和推荐结论。

## 2. 根因分析

Prompt：

```text
一个插件第一次打开工具时能生效，但重开后就失效。先不要改代码，继续追问为什么，直到找到机制层根因和一个最小验证步骤。
```

预期效果：

- RAVO 应优先进入根因分析，而不是马上给修复方案。
- 回答应区分现象、近因和机制根因。

## 3. 验收门禁

Prompt：

```text
我刚做完一个文件上传功能，代码写完了，单元测试也过了，但还没做真实端到端验证。现在能不能说这个版本可发版？
```

预期效果：

- RAVO 应阻止或降级这个发版结论。
- 回答应指出缺少哪些证据，而不是直接说可发版。

## 4. 小改动

Prompt：

```text
把按钮文案从 Save 改成 Confirm。
```

预期效果：

- RAVO 不应强行走重型分析流程。
- 工具应把它当成一个小而直接的任务处理。
