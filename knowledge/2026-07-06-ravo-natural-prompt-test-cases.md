# RAVO 自然语义触发测试用例

日期：2026-07-06

## 目标

验证 RAVO 是否能在用户自然表达需求、问题、根因和验收诉求时自动进入对应能力，而不是要求用户显式说“调用 RAVO”。

## 预期

- 用户不需要提到 `ravo`、`skill`、`plugin`。
- 需求/方案/架构类 prompt 应触发 `ravo-requirement-analysis` 或等价分析行为。
- bug/根因/为什么类 prompt 应触发 `ravo-root-cause-analysis` 或等价根因行为。
- 验收/发版/上线类 prompt 应触发 `ravo-release-acceptance` 或 acceptance gate。

## 测试 Prompt

### 1. 需求洞察

```text
我们正在做一个 AI 穿搭小程序，用户上传衣服照片后，系统自动生成衣橱标签。现在我想新增“旅行场景穿搭推荐”：用户输入目的地、天数、天气和行李箱大小，系统推荐每天穿什么。这个需求先别开发，帮我判断真正的用户是谁、目标是什么、有哪些边界和风险，然后给出推荐方案。
```

期望：输出目标、消费者、约束、选项、推荐方案和验证方式。

### 2. 方案分析

```text
我们准备把 Agent 生命周期治理能力开源，包含需求分析、任务编排、验收证据和知识沉淀四个模块。我不确定应该发布为一个大而全插件，还是发布成多个统一品牌的独立插件。你先从采用成本、扩展性、维护风险和用户心智分析。
```

期望：不直接编码；先做方案权衡和结论推导。

### 3. 根因深挖

```text
我们的验收插件现在能拦截“可以验收了吗”，但对“这个版本是不是能发”没有触发。我觉得这可能只是关键词覆盖问题，也可能是设计模式问题。请继续追问为什么，直到找到可验证、可防复发的机制根因。
```

期望：包含 symptom、proximate cause、mechanism root cause、Why chain、验证方式。

### 4. Bug 根因

```text
一个 Codex 插件在空 workspace 里偶发把 release 状态判断成 ready，但实际没有 screenshots、API evidence，也没有 acceptance artifact。这个 bug 先不要改代码，先分析 symptom、proximate cause、mechanism root cause。
```

期望：不急于改代码；先分层定位根因和最小验证。

### 5. 验收判断

```text
我刚完成了积分扣减功能：新人赠送 20 分，AI 入橱扣 1 分，AI 试穿扣 5 分，失败要退款。代码已经写完，跑了单元测试，但还没做真实小程序端到端验证。这个功能可以验收了吗？
```

期望：如果缺少 `knowledge/.ravo/` acceptance artifact 或证据不足，自动阻塞或明确指出缺口。

### 6. 发版判断

```text
现在准备发布衣橱管理 v1.2.0：后端接口已部署，前端页面已打包，小程序开发者工具能打开首页，但还没有真机截图，也没有上传衣服到生成标签的完整链路记录。现在能不能说这个版本可发版？如果证据不够，请直接指出缺哪些证据。
```

期望：状态语言与证据匹配；不能用口头总结替代验收证据。

### 7. 上线状态

```text
我们把 RAVO v0.1 推到了 GitHub，README 能打开，三个插件在本机已安装启用，但普通 git push 还因为网络问题不可用，远端是通过 GitHub API 写入的。这个版本是不是已经可以说已上线？请按证据判断，不要按主观感觉判断。
```

期望：触发 acceptance gate；没有部署和真实路径证据时不得声称已上线。

## 判定标准

- 通过：用户未显式提到 RAVO，但 Codex 自动采用 RAVO 的结构、约束或 gate。
- 部分通过：Codex 没有明确调用 skill，但输出满足 RAVO 结构。
- 失败：Codex 直接泛泛回答、跳过根因链、或在证据不足时给出验收/发版/上线结论。
