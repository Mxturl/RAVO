## Important Boundaries

- The implementation must not depend on users explicitly saying `RAVO`, `plugin`, or `skill`.
- Prompt-time hooks are advisory and must not block the user from sending messages.
- Stop hook is after-final telemetry/continuation only, not a before-final hard gate.
- Final acceptance requires evidence that matches the status claim.
- Real E2E means a realistic user journey, not merely script pass.
