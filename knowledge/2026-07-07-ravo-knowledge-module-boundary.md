# RAVO knowledge 模块边界

日期：2026-07-07

## 结论

RAVO 当前没有完整的 knowledge 管理闭环。v0.1.2 只具备 workspace-local artifact 写入和 manifest 发现能力，不应宣称已经具备长期知识管理能力。

更准确的产品边界是：原始项目事实、客户需求、代码约定和验收证据默认只属于当前 workspace；从项目中抽象出的经验、原则和反复有效的方法可以迁移到用户级知识层，但必须脱敏、标注适用边界，并由用户显式 opt-in。

后续如实现 `ravo-knowledge`，必须同时满足：

- 写入：沉淀项目内事实、决策、约束、失败教训和验证证据；可迁移内容必须另行抽象为 lesson/principle。
- 检索：在新任务开始前，检索当前 workspace 知识；用户 opt-in 后，再检索用户级可迁移 lessons。
- 应用：在输出或执行计划中显式说明采用了哪些知识、哪些知识不适用、是否需要更新。

## 边界

- 原始项目事实只能默认读取和写入当前 workspace 的 `knowledge/.ravo/knowledge` 或项目级 `knowledge/`，不得默认跨项目共享。
- 用户级可迁移知识只能保存脱敏后的 lessons/principles，且必须记录来源、适用边界和用户 opt-in 状态。
- 新项目可以复用用户级 lessons，但不能把旧项目事实当成新项目事实。
- 不把 knowledge 模块做成中心化调度器；它只提供可检索、可应用、可审计的项目知识层。
- 不接受只写 artifact、不在后续任务检索和应用的实现。

## 最小验收标准

- 有可运行的写入命令。
- 有可运行的检索命令，能返回命中的知识条目和引用路径。
- 有任务入口 hook 或 skill 契约，要求 Agent 在中高复杂度任务前检索当前 workspace 知识。
- 有回归测试证明：写入一条 workspace 知识后，同项目相似任务能检索并应用。
- 有回归测试证明：抽象一条用户级 lesson 后，另一个 workspace 能检索到 lesson，但不会读取原始项目事实。
