# RAVO v0.3 memory reference notes

RAVO v0.3 keeps the knowledge layer intentionally small: human-readable Markdown plus a JSON index/artifact layer. This matches the v0.3 constraint to avoid a central dispatcher, heavy storage backend, or silent global memory writes.

Reviewed references:

- Mem0: useful ideas are multi-level memory, agent-generated facts as first-class memory, and multi-signal retrieval. RAVO v0.3 borrows the classification and retrieval discipline, not the managed service or vector/backend architecture.
- Letta/MemGPT: useful idea is long-lived stateful agents with memory. RAVO v0.3 borrows the principle that memory must be reusable across sessions, not the full agent runtime.
- Graphiti/Zep: useful ideas are provenance, temporal context, and changed facts. RAVO v0.3 borrows source/applicability/updated metadata, not a graph database.
- ShadowMatrix: useful direction is human-readable, token-ready personal corpus with categories. RAVO v0.3 maps this into project lifecycle knowledge categories.

Decision:

- Implement workspace-local Markdown knowledge files with frontmatter.
- Maintain `knowledge/.ravo/knowledge/index.json` for retrieval.
- Keep user-level global knowledge opt-in only, with source, sensitivity, applicability, and redaction.
- Do not add vector search, graph search, or external services in v0.3.
