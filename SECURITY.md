# Security Policy

## Supported Version

Security fixes target the latest published RAVO version.

## Reporting

Report vulnerabilities privately through GitHub Security Advisories for `Mxturl/RAVO`. Do not open a public issue containing credentials, private workspace data, exploit details, or user records.

Include the affected version, impact, minimal reproduction, and any safe mitigation. The project will confirm receipt, assess severity, and coordinate disclosure before publishing a fix.

## Data Boundary

RAVO stores workspace artifacts locally by default. Review Provider credentials belong in user configuration outside the repository. User-level transferable Knowledge requires explicit opt-in and redaction.
